# 局域网测试快速参考

## 🚀 一键启动（推荐）

直接双击运行：`启动局域网测试.bat`

这会自动启动后端API和前端Web管理后台。

---

## 📋 手动启动步骤

### 1️⃣ 启动后端API
```bash
cd wanlian-tms-api
npm run dev
```

### 2️⃣ 启动前端（局域网模式）
```bash
cd web-admin
npm run dev:lan
```

---

## 📱 同事访问地址

启动后运行 `获取本机IP.bat`，或查看前端启动窗口显示的地址：

```
http://[你的IP]:5173/
```

例如：
```
http://192.168.1.100:5173/
```

## ⚠️ 常见问题

### 同事无法访问？

**检查防火墙设置**：
1. 按 `Win + R`，输入 `wf.msc` 回车
2. 点击"入站规则" → "新建规则"
3. 规则类型：端口
4. 协议：TCP
5. 端口：5173, 3000
6. 操作：允许连接

**临时关闭防火墙测试**：
- 控制面板 → Windows Defender 防火墙
- 启用或关闭 Windows Defender 防火墙
- 专用网络和公用网络都选择关闭

### 前端能访问，API请求失败？

1. 确保后端也在运行
2. 检查后端窗口是否有错误
3. 同事浏览器控制台会显示具体的API地址和错误

### 如何确认在同一网络？

1. 你和同事连接到**同一个Wi-Fi**
2. 或者网线连接到**同一个路由器**
3. 公司网络需要在同一VLAN
