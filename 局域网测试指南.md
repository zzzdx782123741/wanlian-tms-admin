# Web管理后台局域网测试指南

## 同事如何访问你的Web管理后台

### 步骤1：获取本机局域网IP地址

**Windows系统**：
1. 按 `Win + R` 打开运行窗口
2. 输入 `cmd` 并回车
3. 在命令行中输入：`ipconfig`
4. 找到"IPv4 地址"，例如：`192.168.1.100`

**或者使用PowerShell**：
```powershell
ipconfig | findstr IPv4
```

### 步骤2：启动后端API服务器

后端API需要监听所有网络接口才能被局域网访问。

**检查后端启动命令**：
```bash
cd wanlian-tms-api
npm run dev
```

**如果后端只监听 localhost，需要修改为监听 0.0.0.0**

### 步骤3：启动前端开发服务器（使用局域网配置）

我们已经为你准备好了启动脚本，运行：

```bash
cd web-admin
npm run dev:lan
```

这会自动：
- 监听所有网络接口（0.0.0.0）
- 使用本机局域网IP作为API地址

### 步骤4：告诉同事访问地址

假设你的局域网IP是 `192.168.1.100`，启动后终端会显示：

```
  VITE v5.0.0  ready in 500 ms

  ➜  Local:   http://192.168.1.100:5173/
  ➜  Network: http://192.168.1.100:5173/
```

**同事在浏览器中访问**：
```
http://192.168.1.100:5173/
```

## 注意事项

### 防火墙设置
如果同事无法访问，可能是Windows防火墙阻止了连接。

**临时关闭防火墙测试**（仅测试用）：
1. 打开"控制面板" → "Windows Defender 防火墙"
2. 点击"启用或关闭 Windows Defender 防火墙"
3. 专用网络和公用网络都选择关闭

**或者添加规则允许Node.js访问**：
1. 打开"Windows Defender 防火墙" → "高级设置"
2. 选择"入站规则" → "新建规则"
3. 规则类型：端口
4. 协议：TCP，端口：5173
5. 操作：允许连接

### 确保在同一网络
- 确保你的电脑和同事的电脑连接到**同一个Wi-Fi或路由器**
- 如果是公司网络，可能需要在同一VLAN

## 快速操作流程

### 在你的电脑上：

1. **打开命令行，获取本机IP**
   ```bash
   ipconfig
   ```

2. **启动后端API**
   ```bash
   cd wanlian-tms-api
   npm run dev
   ```

3. **启动前端（局域网模式）**
   ```bash
   cd web-admin
   npm run dev:lan
   ```

4. **将显示的地址发送给同事**
   ```
   http://192.168.1.100:5173/
   ```

### 在同事的电脑上：

1. 打开浏览器
2. 输入你发送的地址
3. 开始测试！

## 常见问题

**Q: 同事访问时显示"无法访问此网站"**
A: 检查防火墙设置，确保端口5173和3000允许入站连接

**Q: 前端能访问，但API请求失败**
A: 检查后端是否也在监听 0.0.0.0，同事的浏览器控制台会显示API地址

**Q: 如何查看本机IP？**
A: 运行 `ipconfig` (Windows) 或 `ifconfig` (Mac/Linux)

**Q: 端口被占用怎么办？**
A: 修改 vite.config.js 中的端口号
